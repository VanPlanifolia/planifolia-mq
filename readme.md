##### **planifolia-mq**

轻量级延迟消息框架，用于简化业务中延迟任务的分发与处理。

##### 💡 设计思路

planifolia-mq 旨在提供一个基于注解驱动的延迟消息机制，核心思想如下：

消费者注册器（ConsumerRegister）
基于注解扫描，将标注的方法注册为消息消费处理器，当前支持通过方法注册，后续将升级为基于类的注册方式，结合接口约定以增强规范性。

分发器（Dispatcher）
根据消息类型或标识，将接收到的消息路由至对应的消费者并触发执行，实现解耦的消息分发模型。

延迟任务封装（DelayTaskWrapper）
使用统一结构封装待执行任务，支持设置延迟时间与执行上下文，便于后续拓展链路追踪、失败重试等功能。

发送模板（DelaySendTemplate）
提供统一的消息发送入口，使用者只需关注业务数据与延迟时间，无需关注底层实现细节。

##### 🔧 后续计划
